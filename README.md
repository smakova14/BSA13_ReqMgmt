# BSA13_ReqMgmt

Проект по управлению требованиями и тестированию для системы онлайн-записи барбершопа (BRB): выделение типов требований, регистрация требований в системе управления задачами, разработка позитивных и негативных тест-кейсов, оценка результатов и создание отчёта о дефекте.

## 🛠 Навыки
- Requirements Management (типы, атрибуты, статусы, трассировка)
- User Story decomposition
- Test Case design (positive & negative)
- Defect reporting
- Task tracking (WEEEK / Trello)

## 📋 Описание проекта
В ходе работы я выполнила полный цикл управления требованиями для фрагмента задачи 1 (автоматическая отправка напоминаний клиентам), включая выделение типов требований, регистрацию их в трекере, создание тест-кейсов и анализ дефектов.

### Что именно было сделано:
1. **Выделение типов требований (ex00)**: Определила 4 группы требований (бизнес-требования, требования пользователей, функциональные, нефункциональные). Назначила идентификаторы, атрибуты (статус, приоритет, версия, ответственные) и статусную модель. Развернула персональный экземпляр WEEEK и зарегистрировала требования.
2. **Создание требований и регистрация (ex01)**: Декомпозировала выделенное предложение задачи 1 на User Stories (информирование менеджером, получение клиентом, каналы связи, настройка расписания). Зарегистрировала функциональные требования и требования к интерфейсам. Указала приоритеты, зависимости и горизонтальные связи.
3. **Позитивный тест-кейс (ex02)**: Разработала тест-кейс для UC02 (создание расписания мастера). Указала идентификатор, название, приоритет, предусловие, шаги, ожидаемый результат, постусловия. Зарегистрировала задачу на тестирование в WEEEK.
4. **Негативный тест-кейс (ex03)**: Разработала негативный тест-кейс для UC02 (попытка создать слот в уже забронированное время). Выполнила аналогичную структуру и регистрацию.
5. **Оценка результата (ex04)**: Проанализировала результаты выполнения позитивного тест-кейса. Для каждого из двух сценариев (система сообщила о бронировании, но клиент отрицает; система закрепила/не закрепила слот) определила результат. В случае отрицательного результата создала отчёт о дефекте с идентификатором, шагами воспроизведения, ОР и ФР, окружением, приоритетом и важностью.
